Introduction to Xamarin.iOS

Duration

15 minutes

Lab Goals

In this optional homework exercise, you will add an additional piece of UI to allow the user to select the tip percentage to apply and then extend the logic to utilize this information to calculate the tip amount.

Required assets

The provided Homework folder contains a subfolder named Completed with a solution you can use to check your work or to see how to implement the logic.

High-level steps

Here is the final UI you will be generating. We have added a UISegmentedControl just after the button to allow the user to select an appropriate tip percentage. This control is similar to a set of mutually exclusive radio buttons.

Here is what you will need to do for this exercise, if you have any trouble or need a hint, check the completed solution included with the lab materials.

  1. Add a UISegmentControl into the UI between the button and the result label. It should be 20pts from the left and right, and have an 8pt margin on the top and bottom like the other controls.
  2. Insert 4 segments into the UISegmentedControl using the InsertSegment method. You want 10%, 15%, 20% and 25%.
    • The method takes three parameters: the text to display, the position of the button (starting at zero), and whether to animate in - this last parameter should be false since we are adding the control during the initial load and not later in the app lifecycle.
    Here's an example of inserting the 10 and 15% segments: 
    3. UISegmentedControl tipAmount = ...;
tipAmount.InsertSegment("10%", 0, false);
tipAmount.InsertSegment("15%", 1, false);
  3. Use the SelectedSegment property to set the initial selection to the 20% segment (should be index "2").
  4. Insert the segment control into the parent view hierarchy.
  5. Finally, update your tip logic to get the selected segment and use the appropriate tip amount percentage.

Bonus Challenge

As a bonus challenge, see if you can get the tip to be recalculated automatically if you change the tip percentage without tapping the Calculate button. This will require a few changes:

  1. You will need to handle the ValueChanged event from your UISegmentedControl.
  2. You will need to move the tip calculation logic into it's own method - this is your Model logic. You can move it into it's own source file, or just into a method in the View Controller since it is very simple.
  3. The tip calculation logic needs access to the UI elements - you will need to either pass the values in, or make the controls available at the class level by promoting them to fields in the view controller.

Summary

In this homework exercise you added some logic to make the tip amount customizable using a UISegmentedControl. Check the completed sample in the Homework folder if you would like to see an example implementation.
